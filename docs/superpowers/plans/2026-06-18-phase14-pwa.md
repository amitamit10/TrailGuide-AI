# TrailGuide AI — Phase 14: PWA & Offline Mode

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make TrailGuide AI installable as a home-screen app on Android and iOS, and let users view their trip timeline while offline — critical for travellers with no data.

**Architecture:** `@serwist/next` (the actively-maintained fork of next-pwa, supporting Next.js 15+) registers a service worker that pre-caches the app shell and runtime-caches API responses for the trip timeline. A `public/manifest.json` makes the app installable. An offline fallback page shows when the user navigates to a route that hasn't been cached. Service workers do not work in `next dev` with Turbopack — always test with `next build && next start`.

**Tech Stack:** `@serwist/next` v9, `serwist` v9, `next build` (required for service worker testing).

## Global Constraints

- Service workers require HTTPS in production. They also work on `localhost` for local testing, but NOT via `next dev` + Turbopack — always test PWA with `next build && next start`.
- Read `node_modules/next/dist/docs/01-app/` before touching `layout.tsx` or `next.config.ts` — Next.js 16 APIs may differ from training data.
- App icons must be real PNG files in `public/icons/`. The plan includes an SVG-to-PNG conversion step using `sharp` (no browser needed).
- Theme colour: `#2D6A4F` (forest green). Background colour: `#FAFAF8` (off-white). These match the existing design system.
- Do NOT enable background sync or push notifications in this phase — that is Phase 7.
- iOS Safari does not support the `beforeinstallprompt` event. The install prompt code only fires on Android Chrome / Edge.

```javascript
// scripts/gen-icons.mjs
import sharp from "sharp";
import { mkdirSync } from "fs";

mkdirSync("public/icons", { recursive: true });

const svg = `<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
  <rect width="100" height="100" rx="22" fill="#2D6A4F"/>
  <text x="50" y="68" font-size="58" text-anchor="middle" font-family="serif" fill="white">✈</text>
</svg>`;

const buf = Buffer.from(svg);

await sharp(buf).resize(192, 192).png().toFile("public/icons/icon-192.png");
await sharp(buf).resize(512, 512).png().toFile("public/icons/icon-512.png");
await sharp(buf).resize(180, 180).png().toFile("public/icons/apple-touch-icon.png");

console.log("Icons generated in public/icons/");
```

Run it:
```bash
export NVM_DIR="$HOME/.nvm" && \. "$NVM_DIR/nvm.sh" && node scripts/gen-icons.mjs
```

Expected: `public/icons/icon-192.png`, `public/icons/icon-512.png`, `public/icons/apple-touch-icon.png` created.

Delete the script:
```bash

```

Expected: packages added, no peer dependency errors.

- [ ] **Step 2: Create `src/app/sw.ts` (service worker source)**

```typescript
import type { PrecacheEntry, SerwistGlobalConfig } from "serwist";
import { Serwist } from "serwist";
import { defaultCache } from "@serwist/next/worker";

declare global {
  interface WorkerGlobalScope extends SerwistGlobalConfig {
    __SW_MANIFEST: (PrecacheEntry | string)[] | undefined;
  }
}

declare const self: ServiceWorkerGlobalScope;

const serwist = new Serwist({
  precacheEntries: self.__SW_MANIFEST,
  skipWaiting: true,
  clientsClaim: true,
  navigationPreload: true,
  runtimeCaching: defaultCache,
  fallbacks: {
    entries: [
      {
        url: "/offline",
        matcher({ request }) {
          return request.destination === "document";
        },
      },
    ],
  },
});

serwist.addEventListeners();
```

- [ ] **Step 3: Update `next.config.ts` to wrap with Serwist**

Replace the contents of `next.config.ts` with:

```typescript
import type { NextConfig } from "next";
import withSerwistInit from "@serwist/next";

const withSerwist = withSerwistInit({
  swSrc: "src/app/sw.ts",
  swDest: "public/sw.js",
  disable: process.env.NODE_ENV === "development",
});

const securityHeaders = [
  { key: "X-Frame-Options", value: "SAMEORIGIN" },
  { key: "X-Content-Type-Options", value: "nosniff" },
  { key: "Referrer-Policy", value: "strict-origin-when-cross-origin" },
  { key: "Permissions-Policy", value: "camera=(), microphone=(), geolocation=(self)" },
  {
    key: "Content-Security-Policy",
    value: [
      "default-src 'self'",
      "script-src 'self' 'unsafe-inline' 'unsafe-eval'",
      "style-src 'self' 'unsafe-inline' https://fonts.googleapis.com",
      "font-src 'self' https://fonts.gstatic.com",
      "img-src 'self' data: blob: https://*.supabase.co https://upload.wikimedia.org https://images.unsplash.com https://lh3.googleusercontent.com",
      "connect-src 'self' https://*.supabase.co https://api.groq.com https://api.telegram.org https://api.tavily.com https://api.unsplash.com https://en.wikipedia.org https://api.open-meteo.com",
      "frame-ancestors 'none'",
    ].join("; "),
  },
];

const nextConfig: NextConfig = {
  serverExternalPackages: ["pdf-parse"],
  async headers() {
    return [
      {
        source: "/(.*)",
        headers: securityHeaders,
      },
    ];
  },
  images: {
    remotePatterns: [
      { protocol: "https", hostname: "lh3.googleusercontent.com" },
      { protocol: "https", hostname: "*.supabase.co" },
      { protocol: "https", hostname: "upload.wikimedia.org" },
      { protocol: "https", hostname: "images.unsplash.com" },
    ],
  },
};

export default withSerwist(nextConfig);
```

> **Note:** If Phase 12 (security headers) has already been executed, `next.config.ts` already has the headers config — only add the `withSerwist` wrapper and imports; do not duplicate the headers.
