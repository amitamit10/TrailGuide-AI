# TrailGuide AI — Phase 31: Structured Logging + Error Tracking

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `log.Printf` in Go with zerolog (structured JSON logs), replace `print()` in Python with structlog, add Sentry SDK to all three services for error aggregation, and add correlation IDs so a single user request can be traced across Go → Python → logs.

**Architecture:** Every request gets a `X-Request-ID` header (UUID, generated by Go middleware if not present). Go passes it to Python as a header. Both services include it in every log line. Sentry receives unhandled exceptions + explicitly captured errors. In Next.js, Sentry captures client-side errors and passes the request ID to Go via the API client.

**Tech Stack:** Go: `github.com/rs/zerolog`. Python: `structlog`. Sentry: `@sentry/nextjs`, `sentry-sdk[fastapi]`, Go `github.com/getsentry/sentry-go`. New env var: `SENTRY_DSN` (same DSN for all 3 services, or one DSN per service — either works).

**Prerequisite:** Phase 19 (all three services deployed).

## Global Constraints
- Sentry DSN is required in production; optional in dev (logs skip Sentry if DSN is empty).
- Correlation ID header name: `X-Request-ID` (UUID v4).
- Log level is controlled by `LOG_LEVEL` env var: `debug`|`info`|`warn`|`error`. Default: `info`.
- Sensitive fields (Authorization header, GROQ_API_KEY) MUST NOT appear in logs.
- New env var: `SENTRY_DSN` (all three services), `LOG_LEVEL` (all three services).

- [ ] **Step 2: Create `backend/internal/middleware/requestid.go`**

```go
package middleware

import (
	"github.com/gin-gonic/gin"
	"github.com/google/uuid"
)

const RequestIDKey = "request_id"
const RequestIDHeader = "X-Request-ID"

func RequestID() gin.HandlerFunc {
	return func(c *gin.Context) {
		id := c.GetHeader(RequestIDHeader)
		if id == "" {
			id = uuid.New().String()
		}
		c.Set(RequestIDKey, id)
		c.Header(RequestIDHeader, id)
		c.Next()
	}
}
```

- [ ] **Step 3: Replace `log.Printf` calls with zerolog in `main.go` and all handlers**

```go
package main

import (
	"os"
	"github.com/rs/zerolog"
	"github.com/rs/zerolog/log"
)

func initLogger() {
	level := zerolog.InfoLevel
	if l, err := zerolog.ParseLevel(os.Getenv("LOG_LEVEL")); err == nil {
		level = l
	}
	zerolog.SetGlobalLevel(level)
	if os.Getenv("LOG_FORMAT") == "pretty" {
		log.Logger = log.Output(zerolog.ConsoleWriter{Out: os.Stdout})
	}
}
```

// In Gin middleware — capture 5xx errors:

	}

import os
import structlog

def configure_logging():
    structlog.configure(
        processors=[
            structlog.contextvars.merge_contextvars,
            structlog.processors.add_log_level,
            structlog.processors.TimeStamper(fmt="iso"),
            structlog.processors.JSONRenderer(),
        ],
        wrapper_class=structlog.make_filtering_bound_logger(
            getattr(__import__("logging"), os.getenv("LOG_LEVEL", "INFO").upper(), 20)
        ),
    )

- [ ] **Step 3: Add request ID middleware to `main.py`**

```python
import os
import uuid
import sentry_sdk
from sentry_sdk.integrations.fastapi import FastApiIntegration
import structlog
from fastapi import Request
from starlette.middleware.base import BaseHTTPMiddleware
from logging_config import configure_logging

configure_logging()
logger = structlog.get_logger()

dsn = os.getenv("SENTRY_DSN")
if dsn:
    sentry_sdk.init(dsn=dsn, integrations=[FastApiIntegration()], traces_sample_rate=0.1)

class RequestIDMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next):
        request_id = request.headers.get("X-Request-ID", str(uuid.uuid4()))
        structlog.contextvars.bind_contextvars(request_id=request_id)
        response = await call_next(request)
        response.headers["X-Request-ID"] = request_id
        structlog.contextvars.clear_contextvars()
        return response

app.add_middleware(RequestIDMiddleware)
```
