# TrailGuide AI — Phase 44: Multi-Language UI (i18n)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Translate the TrailGuide UI into 4 languages: English (default), Spanish, French, and Hebrew (RTL). Language is stored in user preferences and applied via Next.js i18n routing. AI-generated content (itinerary, chat) remains in the detected language of the destination (already handled by Groq).

**Architecture:** Use `next-intl` for translation management. Translation files (`en.json`, `es.json`, `fr.json`, `he.json`) in `messages/`. Locale stored in `profiles.preferences.language`. RTL support for Hebrew via `dir="rtl"` on `<html>`. Language selector in profile settings.

**Tech Stack:** `next-intl` 3.x, Next.js i18n routing.

**Prerequisite:** Phase 18 (pure Next.js frontend). Phase 41 (CSS variables ready for RTL adaptation).

## Global Constraints
- Do NOT auto-detect language from browser — always use stored preference (English default).
- AI-generated content (trip itineraries, chat, culture pack) is NOT translated — it comes from Groq in whatever language is most natural for the destination.
- URL structure: `/en/dashboard`, `/es/dashboard`, `/fr/dashboard`, `/he/dashboard`.
- Translate UI chrome only: navigation, buttons, labels, error messages, empty states, onboarding.
- RTL: Hebrew requires `dir="rtl"` and mirroring of left/right padding — Tailwind's `rtl:` variants handle this.

- [ ] **Step 2: Handle RTL for Hebrew**

In `src/app/[locale]/layout.tsx`:
```tsx
import { routing } from "@/i18n/routing";

export default function LocaleLayout({ children, params: { locale } }) {
  const dir = locale === "he" ? "rtl" : "ltr";
  return (
    <html lang={locale} dir={dir}>
      <body>{children}</body>
    </html>
  );
}
```

Tailwind RTL utilities (add `rtl:` prefix):
```tsx
// Left padding becomes right in RTL:
<div className="pl-4 rtl:pl-0 rtl:pr-4">
// Left-aligned text stays left in LTR, right in RTL:
<p className="text-left rtl:text-right">
```
